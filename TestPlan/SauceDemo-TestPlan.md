# 1. Objective

The purpose of this Test Plan is to verify that the SauceDemo web application functions correctly and meets the specified business and functional requirements.

---

# 2. Scope

## In Scope

- Login
- Inventory
- Shopping Cart
- Checkout
- Logout
- Navigation
- Error Messages
- UI Validation

## Out of Scope

- Performance Testing
- Security Testing
- Automation Testing
- Database Testing

---

# 3. Test Strategy

Testing will be performed manually using black-box testing techniques.

The following testing types will be executed:

- Functional Testing
- UI Testing
- Positive Testing
- Negative Testing
- Boundary Value Analysis
- Error Message Validation
- Regression Testing (if required)

# 5. Test Deliverables

The following documents will be delivered:

- Test Plan
- Test Scenarios
- Test Cases
- Bug Reports
- RTM (Requirement Traceability Matrix)
- Test Summary Report

---

# 6. Entry Criteria

Testing will start when:

- Application is available.
- Test environment is accessible.
- Test data is available.
- Requirements are reviewed.

---

# 7. Exit Criteria

Testing will be considered complete when:

- All planned test cases have been executed.
- Critical defects are fixed or reported.
- Test Summary Report is completed.
- Stakeholder approval is received.

---

# 8. Assumptions

- Application requirements remain stable during testing.
- Test environment remains available.
- Test accounts are accessible.
- Internet connection is stable.

---

# 9. Risks

Potential risks include:

- Website downtime.
- Requirement changes.
- Environment instability.
- Browser compatibility issues.
